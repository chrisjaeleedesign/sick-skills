# Challenge Flow

Stress-test an idea by arguing against it, then steelman the result.

## When to use

Someone (the user or the agent) has taken a position and wants to pressure-test it before committing. Good for architecture decisions, technology choices, or any "I think we should..." moment.

## Steps

1. **Capture the position** — State the idea or approach clearly.
2. **Challenge** — Call ask with a contrarian persona or framing: "Argue against this approach. What are the strongest reasons not to do it?"
3. **Present** — Show the counterarguments to the user.
4. **Steelman** — Call ask again: "Given the original position and these counterarguments, what's the strongest version of the original idea? How would you address the concerns?"
5. **Decide** — The user now has the original idea, the best arguments against it, and an improved version that addresses the concerns.

## Tips

- The challenge step works best with a different model than the one that proposed the idea. Fresh context, no anchoring.
- Don't use this for trivial decisions. Save it for choices that are hard to reverse.
