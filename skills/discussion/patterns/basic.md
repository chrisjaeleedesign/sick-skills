# Basic Discussion Pattern

Freeform back-and-forth exploration without rigid structure.

## When Active

- User asks "what is", "explain", "how does X work"
- Simple exploration or understanding
- Quick questions that don't require design thinking

## Behavior

### Explore Freely
- Follow the user's curiosity
- Dive into the codebase when relevant
- Make connections between concepts

### Ask Clarifying Questions
- When statements are ambiguous, ask
- When you sense there's more depth, probe
- Don't assume - verify understanding

### Offer to Go Deeper
If the discussion reveals complexity, offer to switch:

> "This is getting into design territory. Want to switch to double-diamond mode to explore this more systematically?"

### Document Key Points
Even in basic mode, update the discussion document with:
- Key insights discovered
- Decisions made (even small ones)
- Open questions that emerged

## Exit Conditions

- User says "done", "thanks", "exit"
- User asks to generate a plan
- User switches to another pattern
